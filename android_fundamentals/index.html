Application Fundamentals

There are four components of Android applications:

  • Activities
  • Services
  • Content providers
  • Broadcast receivers

Activities

Activities is a single screen with a user interface. You can build the user interface using XML or Java. Activity has essentially four states:

  • Active or running,
  • Paused, if an activity has lost focus but still visible,
  • Stop, if an activity completely gone. It still run on the background, until your device kill it, if memory low or something else,
  • Finish if an activity are completely gone or got killed.

public class Activity extends ApplicationContext {

    protected void onCreate(Bundle savedInstanceState);

    protected void onStart();

    protected void onRestart();

    protected void onResume();

    protected void onPause();

    protected void onStop();

    protected void onDestroy();
}

Services

Long running operations in the background without a user interface. Services are useful for things like network operations, playing music, etc. Services can be started with Context.startService() and Context.bindService().

A service can essentially take two forms:

  • Started, when application component (Activity) starts it by calling startService(). It will then run in the background undefinitely, even if the Activity destroyed.
  • Bound, when an application component binds to it by calling bindService()

Content Providers

Used to store and retrieve data and make it accessible to all applications. This is the only way to share data across applications. Android has build in content providers already, such as contact, media, etc.

Broadcast Receivers

A component that responds to system-wide broadcast announcements. For example, when the screen turns off, low battery, etc.
